# RITE Foundation Wave

## Goal

Create the first durable RITE scaffold: repository, Rust validation core, seed
fixture, repo-local skills, and TRACKER dependency placement.

## Scope

- Rite, obligation, taboo, sacred-place, calendar, and evidence primitives.
- CLI fixture validation.
- First BANISH/LUCIA/CANON/PORTO/CERES/MAXIM boundary statements.
- Planned publisher boundaries for PROOF, CROP, PEBBLE, and FLETCH.

## Non-goals

- No theology engine or real-world religious prescription.
- No downstream game mechanics, map geometry, economic prices, historical
  interpretation, or canonical identity.
- No RLINE extraction until repeated consumer needs are proven.

## Pulses

| Pulse | Title | Validation |
|---:|---|---|
| 1 | Repo foundation and intake | `cargo fmt --check`; `cargo test`; `cargo run -p rite-cli -- validate fixtures\seed-rite.json` |
| 2 | Rite and obligation schema | Add fixtures that prove evidence-backed sacred cost and obligation. |
| 3 | Taboo, place, and calendar schema | Add fixtures that prove taboo, sacred-place, timing, mourning, and legitimacy pressure. |
| 4 | Consumer probes | Validate BANISH, LUCIA, CANON, PORTO, CERES, and MAXIM fixture slices. |
| 5 | Publisher contract | First FLETCH local registry seed; CROP, PEBBLE, and PROOF boundaries remain planned. Validate with `fletch registry validate --file .fletch\registries\rite-foundation-assets.json`. |
